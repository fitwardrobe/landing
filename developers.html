Developer Documentation: The Private AI Architecture

Explore the engineering principles, data flows, and security decisions that make FitWardrobe the standard for privacy-first fashion intelligence.

FitWardrobe is not a traditional cloud-first web application. It is built as a **Local-First, AI-Augmented Application**. This document outlines our high-level architecture, our approach to data sovereignty, and how we leverage modern AI models while maintaining total user privacy.

High-Level Architecture

Our infrastructure is divided into three distinct layers: the **Client Layer** (where data lives), the **Secure Proxy Layer** (where routing happens), and the **AI Intelligence Layer** (where results are generated).

Core Principles

1. Data Sovereignty

The "Golden Copy" of the user's wardrobe—including high-resolution images, metadata, and tags—is stored exclusively on the user's device using high-performance local storage. We do not maintain a centralized database of wardrobes or outfit combinations.

2. Secure AI Proxying

All AI requests are routed through a hardened **Edge Proxy**. This serves three critical functions:

  • API Key Isolation: Client-side code never touches or exposes provider keys (Gemini, Groq, etc.).
  • Request Sanitization: The proxy ensures that only necessary metadata is passed to AI models, stripping away any potential PII (Personally Identifiable Information).
  • JWT Authentication: Every request is verified against a Supabase JWT, preventing unauthorized use of the AI resources.

3. Ephemeral AI Inference

When an image is processed for background removal or tagging, the AI model operates on an ephemeral buffer. Once the inference (the "answer") is returned to the user, the temporary buffer is purged. No wardrobe content is used for training external models.

Technology Stack

  • Frontend: React & Vite (High-performance PWA)
  • Vision Engine: Google Gemini 2.0 Flash (for fast, accurate clothing recognition)
  • Chatbot Engine: Groq (Llama-based models for ultra-low latency styling advice)
  • Database/Auth: Supabase (PostgreSQL for user preferences, JWT for security)
  • Hosting: Vercel (Edge Functions for global availability)

Frequently Asked Developer Questions

Do you provide a public API?

Currently, our API is internal-only to ensure the highest standards of privacy and rate control. We are exploring a "Privacy-Safe Webhook" system for future integrations.

How is background removal handled?

Background removal is a hybrid process. High-level masks are generated via edge-computing models, ensuring minimal data transfer and maximum speed.

Can I build on top of FitWardrobe?

We are advocates for open fashion standards. If you are a developer interested in privacy-first fashion AI, please contact us at fitwardrobee@gmail.com.
